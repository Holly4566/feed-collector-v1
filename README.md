# FeedHatch / 收藏孵化器 🥚

一个轻量、可爱、游戏化的本地收藏消化小工具。

把那些“以后再看”的文章、视频、笔记、链接放进粮仓。每天消化一条收藏，完成后孵化一只随机小生物。
别让收藏夹继续吃灰，让它们变成你的电子小动物。

## 在线体验

[打开 FeedHatch](https://holly4566.github.io/FeedHatch-v1.0/)

## 这是什么？

FeedHatch 是一个单文件 HTML 小产品，用来解决一个很常见的问题：

> 收藏的时候雄心万丈，收藏之后再也不看。

它不想做成严肃的知识管理系统，也不追求复杂分类、标签、同步、数据库。
它更像一个轻量的“收藏消化游戏”：

1. 把收藏内容加入粮仓
2. 每天抽一条出来投喂
3. 消化后记录一句心得
4. 孵化一只随机生物
5. 慢慢填满你的生物巢穴和图鉴

## 核心功能

### 📥 加入粮仓

支持添加：

* 标题
* 链接
* 正文内容
* 平台类型：B站、抖音、公众号、小红书、其他

适合存放文章、视频、笔记、教程、灵感、待读内容等。

### 🫘 今日投喂

每天从粮仓里挑出一条待消化内容。
用户可以：

* 打开原链接
* 复制链接
* 复制正文
* 查看全文
* 编辑正文
* 完成消化并孵化生物
* 每天最多换几次内容

### 🥚 随机孵化

消化一条收藏后，会孵化随机小生物。

生物有不同稀有度：

* 普通
* 稀有
* 传说
* 隐藏款

连续打卡会提高仪式感，也会影响部分隐藏款出现机制。

### 🪺 生物巢穴

所有孵化出来的小生物都会进入巢穴。
可以点击查看生物卡片，包括：

* 生物 emoji
* 名字
* 稀有度
* 描述
* 本次消化生成的小评论
* 重复生物的加餐提示

### 📖 生物图鉴

可以查看所有可收集生物的图鉴进度。
已收集的生物会点亮，未收集的保持神秘状态。

### 🔥 连续打卡

记录当前连续消化天数和最高连续纪录。
点击连续打卡卡片，可以查看最近的打卡历史。

### 🛖 粮仓管理

粮仓中可以查看所有收藏，包括待消化和已消化内容。
支持：

* 查看详情
* 打开链接
* 复制链接
* 复制正文
* 查看全文
* 编辑并保存正文
* 删除收藏
* 清空粮仓

### 🔁 备份与恢复

支持导出和导入 JSON 备份文件。

导入备份前会显示：

* 即将导入的收藏数量
* 即将导入的生物数量
* 当前本地已有收藏数量
* 当前本地已有生物数量

导入会覆盖当前本地数据，请谨慎操作。

### 🎮 消消乐

重复孵化的小生物可以用于“消消乐”。
集齐足够的重复卡片后，可以消除并获得一条随机小文案。

## 数据存储说明

FeedHatch 是一个纯前端本地应用。

所有数据都保存在当前浏览器的 `localStorage` 中，包括：

* 收藏内容
* 生物数据
* 打卡状态
* 今日投喂状态
* 备份相关数据

这意味着：

* 不需要账号
* 不需要登录
* 不上传服务器
* 不需要后端数据库
* 换浏览器后数据不会自动同步
* 清理浏览器数据可能会导致内容丢失

重要提醒：
在更换设备、清理浏览器缓存、重装浏览器之前，请先使用“导出备份”。

## 技术栈

FeedHatch 当前是一个单文件应用：

* HTML
* CSS
* 原生 JavaScript
* localStorage

没有使用：

* React
* Vue
* 构建工具
* 后端服务
* 数据库
* 外部依赖

## 本地运行

下载或克隆项目后，直接用浏览器打开：

```bash
FeedHatch.html
```

也可以将项目部署到 GitHub Pages，作为一个静态网页使用。

## 适合谁用？

适合这些人：

* 收藏夹里堆了很多“以后再看”的内容
* 想每天轻量消化一点内容
* 不想使用复杂知识管理工具
* 喜欢可爱、游戏化、有反馈感的小产品
* 想用更轻松的方式对抗信息焦虑

## 产品定位

FeedHatch 不是 Notion。
不是稍后读工具。
不是严肃知识库。
不是 AI 总结器。

它更像是一个小小的收藏养成游戏：

> 你消化一条收藏，世界上就多一只奇怪的小生物。
